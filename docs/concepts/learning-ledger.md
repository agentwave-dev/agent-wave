# Learning Ledger

The Learning Ledger is a public, explicit record of lessons learned during waves.

It is not hidden memory. It is a reviewed artifact that explains what changed in future behavior and why.

## Ledger Entries

A ledger entry should include:

- Date
- Lane
- Goal id
- Observation
- Decision
- Future instruction
- Evidence receipt
- Promotion status

## Promotion Path

Useful lessons can stay as local notes, become checklist items, or graduate into skills. Promotion should require review because a bad lesson can cause repeated bad automation.

## Non-Goals

The ledger should not store secrets, private raw logs, customer data, or opaque model memory. It should be short, reviewable, and tied to receipts.

See `templates/learning-ledger-entry.md` and `examples/fake-app/.agent-wave/learning/example-ledger.md`.
