# CodeLanes Positioning

## One-Line Pitch

CodeLanes is a multi-agent auto-build harness for running many coding agents at once without repo chaos.

## 30-Second Pitch

CodeLanes is the operating discipline around coding agents. It gives each run an isolated lane, durable state pack, skill, detached process, completion receipt, trace, and merge gate.

The model is stateless. The lane is stateful.

Instead of relying on chat memory or giant logs, CodeLanes makes parallel coding-agent work repeatable, auditable, and mergeable.

## GitHub Description

Multi-agent auto-build harness. Run coding agents across isolated lanes with state packs, traces, receipts, and merge gates.

## Target Users

- Developers running multiple coding agents across one repo.
- Maintainers who need receipts, traces, and merge gates before accepting agent changes.
- Teams experimenting with supervised subagents for planning, implementation, tests, docs, safety, and review.
- Builders who want durable repo context without private memory or raw log dumps.

## What CodeLanes Is Not

- CodeLanes is not another coding agent.
- CodeLanes is not a model wrapper.
- CodeLanes is not an autonomous merge bot.
- CodeLanes is not hidden memory for agents.
- CodeLanes is not a place to store secrets, raw logs, or private runtime artifacts.

## Naming Note

CodeLanes is the product, repo, and public brand.

Wave remains the protocol inside CodeLanes: a coordinated set of bounded agent runs across one or more lanes.
